# Week 3 – MLS Database Integration

## Overview

In Week 3, the natural language parser developed in Week 2 was connected to the MLS database. The objective was to build a reusable query layer that converts parsed property filters into parameterized SQL queries and retrieves matching properties from MySQL.

This week includes:

- MySQL connection module
- Active listing search (`rets_property`)
- Sold comparable search (`california_sold`)
- Property card formatting for downstream OpenClaw agents

# Part 2 – Active Listing Search

### Objective

Query the `rets_property` table using dynamic property filters generated by the NLP parser.

### Supported Filters

| Filter | Database Column |
|---------|----------------|
| City | L_City |
| Maximum Price | L_SystemPrice |
| Bedrooms | L_Keyword2 |
| Bathrooms | LM_Dec_3 |
| Square Feet | LM_Int2_3 |
| Property Type | L_Type_ |
| Pool | PoolPrivateYN |
| View | ViewYN |

### SQL Construction

Instead of writing one SQL statement for every search scenario, the query is built dynamically based on which filters are available.

Example:

Input

```json
{
  "city": "Irvine",
  "maxPrice": 1500000,
  "beds": 3
}
```

Generated SQL

```sql
SELECT ...
FROM rets_property
WHERE
    L_Status = "Active"
AND L_City = ?
AND L_SystemPrice <= ?
AND L_Keyword2 >= ?
ORDER BY L_SystemPrice ASC
```

Parameterized queries are used throughout the project to improve security and prevent SQL injection.

# Part 3 – Sold Comparable Search

### Objective

Retrieve recently sold comparable properties from the `california_sold` table.

### Search Criteria

- City
- Residential properties only
- Sold within the last N months
- Ordered by most recent closing date

Example SQL

```sql
SELECT ...
FROM california_sold
WHERE
    City = ?
AND CloseDate >= DATE_SUB(CURDATE(), INTERVAL ? MONTH)
AND PropertyType = 'Residential'
ORDER BY CloseDate DESC
LIMIT 50;
```
